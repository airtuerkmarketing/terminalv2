## The headline — D-095 (premise-recon win)

The drafted plan was a `getUser→getSession` swap. **Recon overrode it:** the Vercel project's
`serverlessFunctionRegion` was the default **`iad1` (US East)** while Supabase is **`eu-central-1`
(Frankfurt)** — so *every* Supabase call from a serverless function was a transatlantic round-trip
(~80–100ms), and the folder-tree page makes ~4 of them. `getUser` was just one. The fix —
co-locating functions with Supabase via `vercel.json` `regions: ["fra1"]` — is **zero app-code,
zero security risk**, and speeds up **every** authed route + the signed-URL route at once, ~25× the
`getSession` gain. `getSession` was not pursued (saves ~15ms once co-located, would weaken
`requireAdmin`'s revocation check). Verified on prod (preview was SSO-protected); EU demo users see
an even cleaner win.
